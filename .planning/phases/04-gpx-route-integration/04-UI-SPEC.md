## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding |
| sm | 8px | Gap between gear strip items (`gap-2`) |
| md | 16px | Default intra-section spacing |
| lg | 24px | App grid padding (`p-6`) |
| xl | 32px | Main layout column gap (`gap-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level; not used in cockpit |

Source: `App.tsx` (`p-6`, `gap-8`), `GearStrip.tsx` (`gap-2`). All tokens already in use.

Exceptions:
- MiniMap fixed container: `w-[160px] h-[160px]` — fixed dimension, not a spacing token; required to prevent Leaflet zero-size mount bug (Phase 3 Pitfall 6 guard).
- ElevationProfile fixed height: `h-[120px]` — layout slot, not spacing token.
- GearStrip touch targets: `min-h-[44px] min-w-[44px]` — accessibility minimum; not an 8-point multiple but a touch-target floor.
- ConnectionBanner height: `h-8` (32px) — banner slot, acceptable at this scale.

## Interaction Contract

### Route Loading

- GPX is loaded server-side at engine startup via `RIDEOS_GPX` environment variable.
- No file-picker UI in Phase 4. Deferred to v2 (out of scope per REQUIREMENTS.md and vault/webapp concept).
- On load, engine sends one `{"type": "route_data", ...}` WS message. `useTelemetry` stores this in a `routeDataRef` (not state).
- Subsequent `{"type": "telemetry", ..., "position_m": N, "route_loaded": true}` ticks update position display only.

### ElevationProfile interactions

- No user interaction. Passive display only.
- Empty state: shows flat gray area + "Keine Strecke geladen" label until `route_loaded: true`.
- Loaded state: gradient AreaChart + amber ReferenceLine at `positionM`.
- Animation: `isAnimationActive={false}` on all Recharts Area elements (established Phase 3 pattern).

### MiniMap interactions

- All interactivity disabled (`dragging={false}`, `scrollWheelZoom={false}`, `doubleClickZoom={false}`, `touchZoom={false}`) — established Phase 3 pattern.
- Empty state: CartoDB dark tile at zoom 5, central Europe center + "Keine Strecke" overlay at z-[1000].
- Loaded state: gray Polyline full route + amber CircleMarker at rider position. Overlay hidden.
- `fitBounds` fires once on first non-null `routeCoords` (depend on `coords.length`, not array reference — prevents re-fit on every render).

### Position marker behavior

- ElevationProfile: `ReferenceLine x={positionM}` — single vertical amber line.
- MiniMap: `CircleMarker` at `coords[bisectRight(cumDist, positionM)]` — radius 6, amber fill, full opacity.
- Both markers freeze (position stops advancing) during BLE reconnect; resume automatically.
- When route completes (`position_m >= total_dist_m`): both markers stay at end position; grade returns to 0%.

Phase 4 adds no new UI-visible labels. Route-related state changes are purely visual (chart + map update).

## Phase 4 Specific Constraints

1. **No new layout changes.** The cockpit grid (`grid-cols-[1fr_auto]`), elevation slot height (`h-[120px]`), and MiniMap dimensions (`w-[160px] h-[160px]`) are locked from Phase 3.

2. **MiniMap fixed dimension must not change.** Leaflet calculates container size on mount; any CSS animation or dynamic sizing causes the grey-tile bug (Phase 3 Pitfall 6).

3. **Route data must not enter `useState` at 4 Hz.** Full route arrays (`lats`, `lons`, `elevations_m`, `cum_dist_m`) stored in `useRef`. Only `position_m` (scalar) flows through state ticks. Violating this causes `200 KB × 4 Hz` re-render thrash.

4. **`React.memo` on all leaf components** — established INFRA-01 rule from Phase 3 (`UI-01 03-02` in STATE.md decisions). ElevationProfile and MiniMap must retain `memo()` wrapping after promotion.

5. **`isAnimationActive={false}`** on all Recharts Area elements — established Phase 3 pattern. Do not remove.

6. **CartoDB dark_all tile layer** stays unchanged on MiniMap. No API token, no config change needed.
