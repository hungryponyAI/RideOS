# Preferences

How the user likes to work, communicate, and receive output.

---

- Language: follows user's lead — notes and planning in German, technical discussion in either
- Responses: concise and direct; no trailing summaries of what was just done
- Vault-first: always read the relevant vault note before proposing a design or architecture decision
- MVP discipline: respect the defined phase order; don't suggest features from later phases when working on earlier ones
- No speculative abstractions: build exactly what the current phase requires, no forward-compatibility shims
